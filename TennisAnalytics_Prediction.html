Research Question

The Research Question

This project asks whether publicly available pre-match features can predict ATP match outcomes beyond what official ATP rankings already capture. The approach mirrors an attribution problem: given a strong baseline signal (rankings), does any additional feature add statistically significant lift?

Attribution framing

ATP rankings are the baseline channel. Each additional signal (serve statistics, custom ELO) is tested the same way a marketing analyst would test a new attribution model — does it beat the control?

Methodology

Six Null Hypotheses

Each model isolates one source of predictive signal. Bootstrap significance tests (10,000 iterations) determine whether each model beats the prior benchmark. This structure ensures every claim is independently testable.

# Null Hypothesis (H₀) Model AUC Result
H1 Physical attributes have no predictive power Logistic Regression 0.574 Rejected p<0.001
H2 Serve stats have no independent predictive power LR + Gradient Boost 0.608 Rejected p=0.0005
H3 ATP rankings add nothing beyond physical context Gradient Boost 0.707 Rejected p<0.001
H4 Serve stats add nothing to ATP rankings LR + Gradient Boost 0.706 Not Rejected p=0.715
H5 ELO adds nothing over ATP rankings Logistic Regression 0.716 Rejected p=0.012
H6 Serve stats add nothing when ELO is included LR + Gradient Boost 0.722 Rejected p=0.004
H4 is the pivotal finding

Serve statistics are already priced into ATP rankings. They only add lift once ELO has captured the variance rankings miss. This is a meaningful null result — not a failure.

Feature Engineering

Three Feature Groups

All historical features use .shift(1) to exclude the current match outcome. Data is sorted by player and tournament date before any rolling calculation.

Available before the match begins. No rolling computation required.

  • Height differential

  • Tournament level
  • Tournament importance weight

  • Hand dominance (encoded)
ATP Ranking Features 21 features

Rankings are pre-match public information. Historical rates use .shift(1).

  • Rank differential & log-transformed
  • Rank ratio
  • Is favourite flag
  • Career win rate (expanding)
  • Surface-specific win rate
  • Level win rate (Grand Slam / Masters)
  • Form last 5, 10, 20 matches
  • Head-to-head win rate
  • Days since last match
  • Matches in last 30 days
Serve Profile 9 features

Rolling historical averages of serve metrics. All .shift(1) safe — prior matches only.

  • Serve win % (overall)
  • Ace rate
  • Double fault rate
  • First serve in %
  • First serve win %
  • Second serve win %
  • Break point save %
  • Hold % (service games won)
  • Return win %
Custom ELO system

K=32 overall, K=40 surface-specific. Each player starts at 1,500. After each match, ratings update based on expected vs. actual outcome. elo_combined_prob is a weighted blend of overall and surface ELO win probabilities. All ELO state is computed from prior matches only — no look-ahead.

Model selection for H5 — and why it matters

On the validation set, Gradient Boosting achieved a higher AUC (0.716) than Logistic Regression (0.710). LR was selected anyway, for a specific reason: elo_combined_prob is itself a calibrated sigmoid probability. LR can express that signal directly through a single linear coefficient; GB adds no meaningful nonlinearity on top of an already-calibrated input. Selecting LR here is a modeling choice, not a data-driven one — and that distinction is important to state explicitly. Test AUC: LR 0.716, GB 0.721.

H4 is not a failure — it is a finding

Serve statistics carry real signal (H2 showed this), but that signal is already captured by ATP rankings. Adding serve features on top of ranking features marginally degraded performance (−0.001 AUC, from 0.707 to 0.706) — not significantly (p=0.715), but the direction is negative. Serve style is already priced into where players are ranked. Testing sequentially reveals this; testing jointly would have buried it.

Findings

What the Data Tells Us

Rankings capture most of what's knowable

ATP rankings alone achieve AUC 0.707. 57.8% of gradient boosting feature importance belongs to ranking features. The tour rankings already price in most available public information about player quality and recent form.

Custom ELO adds real signal

elo_combined_prob is the single most important feature (27.3% of LR coefficient mass). It captures player trajectory and surface-specific momentum that point-in-time rankings miss. The lift is modest (+0.009 AUC) but statistically robust (p=0.012). Notably, the model distributes remaining weight across ranking, career, and form features — not a one-variable model.

Serve profile doesn't beat rankings alone

H4 failed to reject. Serve statistics are real predictors (H2 showed AUC 0.608 from serve alone), but they are already embedded in official rankings. They add lift only once ELO has extracted the residual variance rankings miss (H6, p=0.004).

65% may be near the practical ceiling — with caveats

Pre-match public data likely cannot push prediction accuracy far beyond 65% due to inherent variance in individual match outcomes. This is consistent with published tennis prediction literature, though the specific ceiling for this dataset has not been formally estimated. Brier Score 0.215 vs. 0.250 random confirms well-calibrated predictions — not inflated by leakage.

Business framing

This is the same question asked every week in analytics work: does this new data source add lift over what I already have? Here the answer is nuanced — serve stats add nothing to rankings (fail to reject H4) but add something once ELO is in the model (reject H6). Signal interactions matter. Testing them sequentially, not jointly, is how you find the truth.

Summary

Research Conclusions

All six hypotheses resolved with bootstrap significance testing (10,000 iterations). Results are stable across temporal splits and surface segments.

  • H1 Rejected Physical context predicts above chance — age, height, surface, and tournament level achieve AUC 0.574 (p < 0.001 vs. random).
  • H2 Rejected Serve statistics have independent predictive power — rolling serve averages achieve AUC 0.608 on their own (p = 0.0005 vs. H1).
  • H3 Rejected ATP rankings significantly outperform physical context — AUC 0.707, 64.0% accuracy (p < 0.001). Rankings are the dominant signal in the dataset.
  • H4 Not Rejected Serve stats do not add to ATP rankings alone — AUC 0.706 (−0.001 vs. H3), a marginal decrease that is not statistically significant (p = 0.715). Serve signal is already priced into rankings; adding it slightly degrades, not improves, performance.
  • H5 Rejected Custom ELO adds lift over ATP rankings — AUC 0.716, 64.9% accuracy (p = 0.012). ELO captures trajectory and surface form that static rankings miss.
  • H6 Rejected ELO + Serve beats ELO alone and the ATP baseline — AUC 0.722, 65.3% accuracy (p = 0.004). Serve stats finally add lift once ELO residual variance is available.

Honest Accounting

Known Limitations

A model without a limitations section is a model with hidden ones. These are the material constraints on the claims made above.

ELO cold-start (2015 data)

All players begin at ELO 1,500 in January 2015 with no warm-up period. The first 18–24 months of training data have ELO signals converging from the same arbitrary starting point — variance in elo_combined_prob is artificially compressed in 2015–2016. A proper implementation would use 2013–2014 data purely for burn-in before training begins. This likely reduces the noisiness of early training records but does not invalidate test-set results (2024 ratings are fully converged).

Bootstrap samples records, not matches

The 10,000-iteration bootstrap resamples from 5,490 player records, but these are 2,745 matched pairs (one winner, one loser per match). Sampling records independently breaks those pairs and slightly inflates the effective sample size. Reported p-values are approximately correct but may be marginally anti-conservative — p=0.012 for H5 could be closer to p=0.02–0.03 under a match-level bootstrap. Results are directionally robust; the ELO lift remains significant under conservative assumptions.

Serve data coverage gap

Serve statistics have ~98.9% coverage across the dataset. Models using serve features (H2, H4, H6) may train and test on a slightly different pool of records than ranking-only models (H3, H5). If the ~1.1% of missing-serve records are systematically different from the rest (e.g., concentrated in certain tournaments or years), cross-model AUC comparisons are not on perfectly identical test sets. The effect is small but has not been formally corrected.

No hyperparameter optimisation

Gradient Boosting models (H3, H4, H6) use fixed hyperparameters rather than systematic search. Bayesian optimisation or grid search on the validation set may improve performance for these models — particularly H3, which is the primary benchmark. If GB performance improves under optimisation, the ELO lift (H5) and the H4 null result could narrow or shift. This is the most actionable next step for improving the analysis.
