Methodology

Four Decisions That Define the Model

Each decision reflects a constraint on what information is genuinely available before a match is played.

01
Elite Tournament Filter

Only Grand Slams (G), Masters 1000 events (M), and ATP Tour events (A) are included. Davis Cup, Challenger, and Futures-level matches are excluded. Lower-tier events feature inconsistent competition quality and ranking distortions that add noise without adding predictive signal. The filter produces 25,245 matches across 10 years.

02
Elo Rating Computation

Two Elo systems are maintained per player: an overall rating and a surface-specific rating. Starting rating is 1,500. K-factor is 32. After each match, both ratings update using the standard Elo expected-score formula. Five features are derived: overall and surface Elo differentials, overall and surface win probabilities, and a combined probability that blends both systems. All use a one-position shift to exclude the current match.

03
Zero Data Leakage

Elo ratings are updated only after a match completes. The feature used for prediction reflects the rating entering that match, not the rating after it. Without this shift, the current match outcome contaminates the feature. The validation check: every player's first appearance uses Elo = 1,500, the neutral starting value.

04
Temporal Train/Test Split

Training data covers 2015 to 2022 (39,544 records). Validation covers 2023 (5,502 records), used for hyperparameter selection. The test set covers 2024 (5,444 records), evaluated once. No shuffling. The train-to-validation accuracy gap is 2.0 percentage points, indicating minimal overfitting.

Why not random train/test split?

A random split would allow training examples from 2024 to inform predictions about 2023 matches. Elo ratings are time-series data: a player's 2024 rating reflects every match they played before that point. A temporal split is the only honest test of out-of-sample generalization.

Step 2 of 2 · Physical Attributes and Elo Value (Player-Level Dataset)

Do Height and Age Help? Does Elo Add Over Rankings?

Two follow-up questions from the EDA. Both are answered with formal tests on the player-level dataset (50,490 records).

Physical Attributes · Mann-Whitney U Test

Winners and losers are compared on height differential and age differential using a non-parametric test (Mann-Whitney U), which makes no normality assumption. With ~50k records, even tiny effects reach statistical significance — effect size r is the meaningful metric, not the p-value alone.

Variable r (point-biserial) p-value Effect size Verdict
height_diff 0.103 < 0.001 negligible Significant but not useful
age_diff 0.046 < 0.001 negligible Noise

Effect size r < 0.1 is considered negligible by convention (Cohen, 1988). Both variables are statistically detectable only because of the large sample, not because the signal is meaningful. Height and age were excluded from the model: including them actively reduces accuracy (ablation testing confirmed removing physical attributes yields a +0.4 pp accuracy improvement).

Elo vs ATP Ranking Baseline · Bootstrap Hypothesis Test

The ATP ranking baseline uses the same classifier architecture trained on raw ranking features only (rank_diff, rank_ratio, is_favorite, player_rank_log, opponent_rank_log) — no Elo. The bootstrap test (10,000 iterations) asks whether the Elo model's accuracy gain is statistically distinguishable from noise.

Feature Engineering

Five Features,
One Core Principle

The entire model runs on five Elo-derived features. Each one is computed from match history using a one-position shift (.shift(1)) so the current match outcome is never included in the feature that predicts it.

The leakage problem

Without the shift, a player's Elo rating entering match 50 would already include the result of match 50. The model would train on information it could not have in a real prediction scenario. Training accuracy looks inflated; test accuracy collapses. This is data leakage, the most common source of false results in sports prediction models.

Without .shift(1): Leakage Present

Current match outcome included in Elo update before prediction.

# Wrong: Elo already updated for this match for idx, row in df.iterrows(): p_elo = elo_ratings[row['player']] o_elo = elo_ratings[row['opponent']] # Update ratings first... update_elo(row) # ...then use as feature: LEAKAGE df.at[idx, 'elo_diff'] = p_elo - o_elo # Feature includes current match result!

With .shift(1): Zero Leakage

Elo rating entering the match is the feature; update happens after.

# Correct: capture rating BEFORE update for idx, row in df.iterrows(): p_elo = elo_ratings[row['player']] o_elo = elo_ratings[row['opponent']] # Store pre-match rating as feature df.at[idx, 'elo_diff'] = p_elo - o_elo # Then update for next match update_elo(row) # Feature reflects only prior history.

The Five Features

elo_combined_prob Consensus importance: 51.8%

A weighted blend of the overall Elo win probability and the surface-specific win probability. Combines the two signals into a single calibrated estimate of how likely the player is to win this specific matchup on this specific surface. The single most predictive feature across all three models.

elo_win_prob_surface Consensus importance: 17.6%

Win probability derived from the surface-specific Elo ratings only. A player may have a high overall Elo but a weaker clay-court Elo if they rarely play at Roland Garros or Barcelona. This feature isolates that surface specialization signal.

elo_win_prob_overall 15.7% importance

Win probability from the overall (all-surface) Elo system. Used alongside surface probability in the combined feature.

elo_surface_diff 8.0% importance

Numerical difference in surface-specific Elo ratings (player minus opponent). Captures the magnitude of the surface-specific advantage.

elo_overall_diff 6.9% importance

Numerical difference in overall Elo ratings. Captures the magnitude of the all-surface advantage, independent of the probability transformation.

Validation: first-match check

Every player's first appearance in the dataset starts with Elo = 1,500. All five features are computed from this starting value for the first match, confirming no historical data is used before it exists. The model has seen 50,490 records processed through this pipeline with no leakage warnings.

Why five features outperform a larger feature set here

A larger feature set (ATP rankings, career win rates, recent form windows) was evaluated. The Elo-only specification performed comparably with substantially less complexity. Elo already encodes accumulated performance against quality opponents, surface history, and form by design. Redundant features that replicate Elo signals do not improve a logistic model and add overfitting risk in tree-based models.

Why Logistic Regression wins on five Elo features

The Elo win probability is already a calibrated sigmoid transformation of the rating difference. A logistic model is the natural complement to this input: it learns a linear combination of near-calibrated probabilities. Random Forest and Gradient Boosting add tree structure that is not needed when the input-output relationship is close to linear, and they introduce slight overfitting as a result.
