ATP Tennis Match Prediction: A Progressive Feature Study¶

Can we beat ATP rankings with ELO ratings and serve statistics?¶

Author: Thomas Neira | Date: January 2026
Dataset: ~25,000 elite ATP matches (2015–2024) — Grand Slam, Masters 1000, ATP Tour only

Research Design¶

Overarching Question¶

Can publicly available pre-match data — beyond official ATP rankings — predict match outcomes with statistically significant accuracy?

Six Null Hypotheses Tested in Sequence¶

Each hypothesis builds on the last. Rejection means the new signal layer adds real predictive value.

# Null Hypothesis (H₀) Test Rejection means...
H1 Physical attributes + context have no predictive power LR vs 50% baseline Even basic context lifts above chance
H2 Historical serve statistics have no independent predictive power LR/GB on serve only vs H1 Serve skill predicts outcomes independently of rank
H3 ATP rankings add nothing over physical attributes GB on ranking features vs H1 (bootstrap AUC) Official rankings contain real signal
H4 Serve statistics add nothing to ATP rankings LR/GB on ATP+serve vs ATP-only (bootstrap) Serve quality is not fully captured by rank
H5 ELO ratings add nothing over ATP rankings LR on ELO features vs ATP-only (bootstrap) ELO encodes signal rankings miss
H6 Serve features add nothing when ELO is already included LR/GB on ELO+serve vs ELO-only (bootstrap) Serve style is not fully mediated through ELO

Methodology Principles¶

  • Zero data leakage: All historical features computed with .shift(1) — no current-match information ever used
  • Temporal validation: Train 2015–2022 → Validate 2023 → Test 2024. Never shuffle time.
  • Bootstrap significance: 10,000-iteration AUC-based bootstrap for all model comparisons
  • Consistent split: All six models evaluated on identical train/val/test records

EDA Takeaways¶

Raw match-level serve correlations are near zero (r ≈ 0.01–0.03) — this is expected, not a failure.
At the match level, the winner always outperforms the loser in that specific match by definition.
The useful signal comes from historical serve averages computed across prior matches with .shift(1).

Ranking differential dominates raw correlation analysis, confirming that player quality is the primary predictor.
However, ATP rankings are a weekly snapshot — ELO (updated match-by-match) and serve style may capture
dimensions that a single rank number misses. Sections 3.5 and 3.6 test this directly.

Section 2: Feature Engineering¶

Three feature layers are constructed here, each requiring careful temporal discipline to prevent data leakage:

  1. ELO ratings — custom Elo system updated after each match; pre-match rating captured before the update
  2. ATP-derived features — ranking differentials, form windows, H2H records — all using .shift(1)
  3. Serve profile — rolling historical serve percentages (9 metrics) using .shift(1)

All features are computed on the full dataset before the train/test split, which is correct because:

  • ELO: updated only after each match chronologically, so pre-match ratings are already leakage-free
  • Rolling features: .shift(1) guarantees only prior-match data is used
  • The temporal split at the end ensures no test-period data ever influences training

ELO Rating System¶

ELO assigns each player a numerical strength rating, updated after every match:

  • K=32 for overall rating — controls how fast ratings change
  • K=40 for surface-specific rating — higher K because players have less data per surface
  • Cold start: New players begin at 1,500 (tour average). This is a known limitation — debut matches use average ratings regardless of actual quality.
  • Pre-match capture: The rating before the match is stored as the feature. The update happens after.
  • Combined probability: elo_combined_prob = (overall_win_prob + surface_win_prob) / 2

Section 3: Progressive Model Building¶

Six models, each testing one null hypothesis. The design mirrors an incrementality study: start with the weakest possible predictor, then add feature layers and test whether each adds significant lift.

Structure per hypothesis:

  1. State H₀ and feature set
  2. Train model(s) on training split
  3. Evaluate on 2024 test set (accuracy + ROC-AUC)
  4. Bootstrap significance test vs the appropriate baseline

All models share the identical temporal split from Section 2. Variables produced here
(y_proba_h1 through y_proba_h6) are used in Sections 4–6.

H1 — Context-Only Baseline¶

H₀: Physical attributes (age, height) and tournament context (surface, level, round) have no predictive power beyond random chance.

Feature set: physical_features — age, height, surface encoding, level, round, best_of
Model: Logistic Regression
Significance test: One-sided binomial test vs 50%

  ROC-AUC       : 0.574  (vs 0.500 baseline)
  p-value vs 50%: 0.0000
  → REJECT H1: Physical context has statistically significant predictive power

H₀: Historical serve statistics have no predictive power when used without ATP rankings or ELO.

Research Question Answers¶

Overarching question: Can publicly available pre-match data — beyond official ATP rankings — predict match outcomes with statistically significant accuracy?

Answer: See H5 and H6 bootstrap results above. The key question is whether p_h5 < 0.05.

Hypothesis Summary (populated at runtime)¶

The code cells in Section 3 print specific p-values and CI bounds for each test. The table below is a reading guide — actual results are in the cell outputs.

# H₀ Feature set Benchmark Reject if
H1 Context has no power Physical + context 50% random p_h1 < 0.05
H2 Serve has no independent power Serve only H1 model p_h2 < 0.05
H3 Rankings add nothing over context ATP features H1 model p_h3 < 0.05
H4 Serve adds nothing to rankings ATP + serve H3 model p_h4 < 0.05
H5 ELO adds nothing over rankings ELO features H3 model p_h5 < 0.05
H6 Serve adds nothing to ELO ELO + serve H3 + H5 p_h6_atp / p_h6_elo < 0.05

Design note: H3 (ATP rankings) is the primary benchmark because rankings are the "free" baseline any analyst has access to. A model is only valuable if it significantly clears that bar.

Business Impact Context¶

This analysis mirrors a common problem in marketing analytics and data science: incrementality testing.

The question "does ELO add lift over ATP rankings?" is structurally identical to:

  • "Does this retargeting campaign add conversions beyond organic intent?"
  • "Does this new data signal improve attribution beyond last-click?"
  • "Does adding behavioural data improve churn prediction over CRM features alone?"

In each case, the methodology is the same as used here:

  1. Establish the incumbent baseline (ATP rankings / last-click / CRM only)
  2. Build the challenger model with the new signal added
  3. Bootstrap the AUC difference on held-out data to confirm significance before investing
  4. Report lift with confidence intervals, not just point estimates

Model limitations to communicate to stakeholders:

  1. No bookmaker odds: The Kelly simulation uses model probabilities as proxy odds. Real edge
    requires comparison to closing-line prices — the simulation demonstrates calibration quality only.
  2. Cold-start ELO: Players making their debut receive Elo=1,500 regardless of true strength.
    This degrades accuracy for matches involving newcomers or comeback-from-injury first matches.
  3. Serve stat coverage: ~1.1% of matches have missing serve data, filled with tour medians.
    This introduces slight regression-to-mean bias in the H6 model for those matches.
  4. Temporal stability: All models trained on 2015–2022. If playing styles or competition
    dynamics shift substantially, recalibration against more recent data would be appropriate.
